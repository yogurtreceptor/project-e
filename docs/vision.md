# Vision

Operation Eddy is a local-first Personal Operational Intelligence Platform.

Its long-term purpose is to become the central source of truth for meaningful personal information. Stage 1 is deliberately narrower: build the structured information platform first.

## Stage 1 Focus

Stage 1 stores, organises, displays, searches and connects real-world information.

The platform is built around:

- canonical entities
- first-class relationships
- multiple views over the same local data
- simple, maintainable implementation choices

The first domains are:

- People
- Organisations
- Locations

## Product Principles

- One canonical record should represent each real-world object.
- Relationships should be stored, edited, displayed and navigated directly.
- Local data should remain useful without network access.
- Views should reveal the same underlying data in different practical ways.
- The platform should stay understandable enough to maintain over time.

## Stage 1 Exclusions

Stage 1 does not include AI, chat, dispatcher behaviour, decision support, scheduling, login, WAN features or automation.

Simple import and export tools are allowed when they directly help populate, migrate or protect local data.
